# Code Guidelines

## Git

You are never allowed to use git. This includes any git operation without exception: committing, staging, pushing, pulling, branching, checking out, deleting, resetting, rebasing, merging, or any other git command. Do not run `git` at all.

## Python Packages

Always use the latest stable version of any Python package when adding it to `requirements.txt`. Pin the exact version (e.g. `package==x.y.z`). Verify the latest version from PyPI before writing.
